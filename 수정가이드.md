# 홈페이지 내용 수정 가이드

내 홈페이지에서 어떤 내용을 바꾸고 싶을 때, 어느 파일을 열어야 하는지 정리한 문서입니다.

---

## 1. 기본 원칙

- 이 사이트는 **Jekyll**로 만들어졌고, `_data/*.yaml` 파일에 있는 정보를 읽어서 화면에 자동으로 뿌려줍니다.
- 즉, 대부분의 정보는 **HTML을 직접 건드릴 필요 없이 yaml 파일의 값만 바꾸면** 됩니다.
- 수정 후에는 아래 3줄만 실행하면 실제 사이트(`https://minsun3054document.github.io`)에 1~2분 내로 반영됩니다.

```bash
git add -A
git commit -m "내용 수정"
git push
```

---

## 2. 파일별로 뭘 고칠 수 있나

| 파일 | 여기서 고치는 것 |
|---|---|
| [`_data/main_info.yaml`](_data/main_info.yaml) | 이름, 직함(title), 이메일, 프로필 사진 경로, GitHub/블로그/SNS 링크 |
| [`_data/experience.yaml`](_data/experience.yaml) | 학력·경력 타임라인 (회사/학교, 기간, 직함, 설명) |
| [`_data/publications.yaml`](_data/publications.yaml) | 논문/출판물 목록 |
| [`_data/projects.yaml`](_data/projects.yaml) | 프로젝트 목록 (현재 index.html에서 비활성화된 상태) |
| [`index.html`](index.html) | Bio(자기소개) 문단, Projects 카드, Awards & Honors 목록 — yaml이 아니라 HTML로 직접 적혀있음 |
| `assets/profile-pics/profile.jpg` | 프로필 사진 파일 자체 교체 |
| `assets/cv/cv_web.pdf` | CV PDF 파일 자체 교체 |

## 3. `_data/main_info.yaml` — 이름/연락처/링크

```yaml
name: "Minsun Lee"
title: "Undergraduate Student, Inha University"
email: "minsun3054 [AT] gmail.com"
profile_pic: "/assets/profile-pics/profile.jpg"
github: "https://github.com/minsun3054document"
blog: "https://minsunstudio.tistory.com/"
google_scholar: ""
linkedin: ""
twitter: ""
bsky: ""
```

- 값이 `""` (빈 문자열)로 되어있는 항목(google_scholar, linkedin 등)은 **사이트에 아이콘 자체가 안 뜹니다.** 링크를 넣으면 자동으로 아이콘이 생깁니다.
- `title`은 학위 과정이 바뀌면 (예: 대학원 진학) 여기서 수정하면 됩니다.

### 이메일을 여러 개 쓰고 싶다면
현재 구조는 `email` 값 하나를 그대로 화면에 텍스트로 출력하는 방식이라, **가장 간단한 방법은 한 줄에 같이 적는 것**입니다.

```yaml
email: "minsun3054 [AT] gmail.com / minsunlee [AT] korea.ac.kr"
```

이렇게만 해도 두 이메일이 나란히 표시됩니다. (스팸봇 방지를 위해 `@` 대신 `[AT]`로 적는 관례를 따르고 있어요.)

만약 두 이메일을 줄바꿈해서 보여주거나 각각 클릭 가능한 링크로 만들고 싶다면, `_layouts/default.html`도 같이 손봐야 해서 코드 수정이 필요합니다 — 이건 저한테 요청해주시면 바로 해드릴게요.

## 4. `_data/experience.yaml` — 학력/경력

```yaml
- place: "회사 또는 학교 이름"
  time: "2024 - 2025"
  title: "직함 (예: Research Intern, M.S. Student)"
  subtitle: "설명, 지도교수 등. <br/> 로 줄바꿈 가능"
  category: "work"   # 회사/연구소 = "work" (타임라인 왼쪽), 학교 = "school" (오른쪽)
```

- 새 경력을 추가하려면 이 형식의 블록을 위/아래에 하나 더 추가하면 됩니다.
- 순서 = 화면에 나열되는 순서이니, 최신순으로 정렬하는 게 일반적입니다.
- `category`를 `work`/`school` 둘 중 하나로 정확히 써야 타임라인 좌우 배치가 제대로 됩니다.

## 5. `_data/publications.yaml` — 논문

```yaml
papers:
  - title: "논문 제목"
    authors: "<b>Minsun Lee</b>, Co-author Name"
    venue: "학회/저널 이름, 연도."
    paper_pdf: ""        # 외부 URL 이나 /assets/publications/파일명.pdf
    selected: y           # y로 하면 "Selected" 탭에도 노출
```

- 본인 이름을 `<b>...</b>`로 감싸면 목록에서 굵게 표시됩니다.
- `selected: y`가 없으면 "All" 탭에서만 보입니다.

## 6. `index.html` — Bio / Projects 설명 / Awards

이 세 섹션은 yaml이 아니라 HTML로 직접 적혀 있어서, 파일을 열어 해당 `<div class="docs-section" id="...">` 블록 안의 텍스트를 직접 고치면 됩니다.

- **Bio**: `id="bio"` 블록 안 `<p>` 문단 수정
- **Projects**: `id="projects"` 블록 안, `<div class="paper">` 하나가 프로젝트 하나. 복사해서 새 프로젝트 추가 가능
- **Awards & Honors**: `id="awards"` 블록 안 `<li>` 항목 추가/삭제

## 8. 수정 → 반영까지 전체 흐름 요약

1. 위 표에서 원하는 파일을 찾아서 연다 (VS Code에서 직접 열어도 되고, 저에게 "이 부분 바꿔줘"라고 시켜도 됨)
2. 내용 수정 후 저장
3. 터미널에서:
   ```bash
   git add -A
   git commit -m "수정 내용 설명"
   git push
   ```
4. 1~2분 기다린 후 `https://minsun3054document.github.io` 새로고침해서 확인

**Tip**: 파일을 직접 만지는 게 번거로우면, 그냥 저한테 "경력에 OO 추가해줘", "이메일 하나 더 넣어줘" 처럼 말로 시키셔도 됩니다. 제가 파일 수정부터 커밋·push까지 해드려요.
